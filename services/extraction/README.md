# Extraction service boundary

Phase 2 owns lossless immutable source versions, closed-vocabulary/mock extraction, typed
`TradingIdeaCard` records, and deterministic research memos. Source URLs are provenance only and are
never fetched by this phase.

Supplied text must be nonblank but is otherwise stored byte-for-byte. URL-only requests omit
`raw_text` and never create an extraction request or queue job. `supplied_at_utc` is generated by
PostgreSQL and appears only in source-version output. The six public evidence fields use distinct
Pydantic enum/evidence contracts; their vocabularies are not interchangeable.

The model-facing contract is deliberately narrower than the public card: an optional future LLM may
return only classifications and immutable source-segment references. Server code reconstructs exact
quotes and never asks a model for an instruction, recommendation, size, or executable rule. Local and
CI behavior uses the deterministic mock extractor and needs no credential.

No strategy mapper, provider adapter, backtest, signal, risk approval, broker, paper-order, or live
execution behavior belongs here.
